School Projects

This is a collection of the most useful/interesting/challenging school projects I've worked on. They're broken up by course. A brief description, as well as any important skills I got from each project, are listed.

Hosting school projects publicly on GitHub or any other site isn't allowed, so if any potential employers would like access to these projects, send me an email.

CMSC131: Object-Oriented Programming I (Fall 2018)

Name Poker Hand Evaluator
Language Java
Description An evaluator for hands of Poker. The GUI and underlying program design were provided by the instructor. The methods I wrote are responsible for evaluating the hands of the six players and determining what kind of hand (pair, straight, flush, etc.) they have.
Valuable Skills Basic programming logic. Control structures, looping, dealing with arrays, etc. This project essentially tested your mastery of the basics of programming.

Name Media Rental Manager
Language Java
Description This program manages the inventory and customers of a media rental store, such as BlockBuster. Customers can have different plans which allow them to rent out and return both movies and albums.
Valuable Skills Object-oriented design. Use of inheritance, polymorphism, etc.

CMSC216: Introduction to Computer Systems (Fall 2019)

Name Linked List Implementation II
Language C
Description Another linked list implementation. This one is a doubly-linked list with a dummy head, and it was done in C rather than Java. This was my first major project that was not done in Java.

Description A program that simulates a simplified version of the Unix filesystem. In this case, a filesystem consists of directories, and in each directory, there can be files as well as additional directories. Various Unix commands are simulated, including touch, mkdir, cd, ls, pwd, and rm.

Name pthreaded wc Implementation
Language C

CMSC330: Organization of Programming Languages (Fall 2020)

Name WordNet
Language Ruby
Description An implementation of a WordNet in Ruby. A WordNet is essentially a way of associating words and concepts. It is a database of language and the meanings of words. See the link for a more in-depth explanation.
Valuable Skills Quickly familiarizing myself with a topic I'd never heard of before (WordNets) while simultaneously learning a language I'd never used before (Ruby).

Name SmallC Lexer, Parser, and Interpreter
Language OCaml
Description This project involved creating an interpreter for a simple language, SmallC. The lexer converts SmallC code into a sequence of tokens, while the parser converts the tokens into an Abstract Syntax Tree (AST). The interpreter will then produce the correct output of the program.

Compilers were briefly covered in the previous class, CMSC216, but this was my first time getting a hands-on look at programming langauges under the hood. This is one of my favorite school projects that I worked on, and it's also the project that finally gave me an appreciation for functional programming.
Valuable Skills Advanced functional programming and a deeper understanding of how compilers work.

CMSC433: Programming Language Technologies and Paradigms (Spring 2021)

Name Restaurant Simulation
Language Java
Description This program simulates the operation of a restaurant. Customers, cooks, and the machines that make the food are all separate, threaded actors that implement the Runnable interface. Customers enter the restaurant and place an order. The order is taken by cooks and processed using the machines, before finally being returned to the customer by the cooks. The system handles numerous customers, cooks, and machines all operating simultaneously.
Valuable Skills Java multithreading, including synchronized blocks, semaphores, etc.

Name Akka Resource Management System
Language Java
Description A program that uses the Akka library to manage actors and their abilities to access resources in a simulated distributed environment. Resources are typically something like a printer, and actors can access and use them using message-passing.
Valuable Skills Use of Akka and general actor-based concurrency concepts.

Name Conway's Game of Life with MPI
Language C
Description This program uses the OpenMPI library to create a parallelized version of Conway's Game of Life. Each row of the grid space is run by a different processor, and they use message-passing to communicate data with one another. I used pthreads for C multithreading in an earlier project, but this was my first experience with MPI.
Valuable Skills Use of the OpenMPI library.

CMSC420: Advanced Data Structures (Spring 2021)

Name Traveling Salesperson Tours
Language Java
Description A program that manages tours for the Traveling Salesperson Problem. Three primary data structures are used. First, the tour itself is simply stored in an ArrayList, though it's in a class which features several utility methods. A slightly modified AA tree is used to map location names to their index within a tour, which makes looking them up more efficient. Finally, the spatial data, the actual coordinates at which locations are located, is stored in a Wrapped K-D Tree.

The Tour and WKDTree classes were written by me. The AAXTree class was provided by the instructor. Implementation of the AA tree was an early part of the assignment, and mine had some errors. The instructor provided canonical implementations after each stage of the project, so I used that rather than spending time trying to fix my version.

This was a very complicated project, by far the most complicated data structure implementation I've ever done. More details about what exactly this project entailed can be found here.
Valuable Skills Advanced data structure implementation.

CMSC411: Computer Systems Architecture (Fall 2021)

Name Pipelined Processor Simulator
Language C
Description A simulation of a basic processor with a simple five-stage pipeline. It runs a simplified, integer-only MIPS instruction set. The program simulates the registers, as well as both instruction and data memory, with the program producing actual output based on what the instructions do. It has a dependency-checking system, which appropriately stalls the pipeline when necessary.

I think it's safe to say this is the most complex project I'd worked on up to this point, at least in terms of what it set out to do. It was an exceptionally challenging project, but also very rewarding, as it not only honed my programming skills, but also expanded my understanding of how a processor works.
Valuable Skills Deeper understanding of how processors work under the hood.

CMSC425: Game Programming (Spring 2022)

Name Mob Hunt
Language C# (in the context of Unity development)
Description A group project in which I worked with one other person to create a video game called Mob Hunt. In the game, the player traverses levels and sets off traps in order to kill enemies. There are three kinds of enemies, each of which behave somewhat differently. There is a fairly robust AI system, in which enemies can move between patrol, pursuit, and search states.

It should be noted that all art assets came from Kenney, a website that provides free assets for games. I am not an artist. As mentioned, this was a group project. One of the things I was responsible for was creating animation for those 3D models. This was done with a simple sine wave moving limbs back and forth, resulting in Minecraft-esque animations. I also implemented player movement, camera movement (which includes a system for turning objects between the player and camera transparent), enemy AI, minor parts of the interface, and one of the two levels. All other features, such as creation of the game concept itself, interaction with traps, most of the UI, the player and enemy health systems, etc., were done by my teammate.

I don't think the game ended up being particularly impressive, but it was definitely a valuable learning experience. We struggled with the concept for the game, so it ended up changing many times throughout the development cycle. If we'd known what we wanted to do from the start, the game may have turned out better.
Valuable Skills C# programming, Unity development, working on a team, and project planning.

CMSC435: Software Engineering (Spring 2022)

Name Online Casino
Language Python (using Django framework), HTML, JavaScript
Description By far the most useful project during my time at UMD, this was a group project in which I developed a web app along with four other people. We followed Agile principles while developing the app, including three sprints and roughly twice-weekly scrum meetings (daily scrum meetings weren't really feasible due to the scale of the project and people's busy schedules). The first sprint was one week, while the subsequent sprints were two weeks each, creating a five-week development cycle. This was perhaps the most valuable part of the project to me, learning how software is actually developed in the industry. This class also taught me how to make my code more readable and how to use Git to collaborate, both of which were applied during the development of this project.

The project itself was Online Casino, a website in which users can deposit and withdraw "money" in order to gamble in five casino games â€” Poker, Blackjack, Craps, Roulette, and Slot Machines. The first three featured fully-functional multiplayer, while the latter two were single-player only. Multiplayer was implemented using websockets, something I had never used before. They ended up being a huge struggle to wrap my head around. Users can sign up, create accounts, become friends with other users, send links to one another to easily join each other's games, and chat with one another during the games. There is a spectating system so people can watch the games unfold without participating. Other features include tutorials for each game and a system to limit the amount of money that can be deposited onto an account with the site each month.

As this was a group project, obviously I was not responsible for everything. One member of our group had already used Django before, so we tended to follow his lead for just about everything. I was responsible for implementing the depositing and withdrawing money systems, the tutorials, the chat system, and the game of Craps.

As mentioned, I think this was the most valuable project I worked on during my time at UMD. This class and project gave me more exposure to real software engineering practices than every other class combined.
Valuable Skills Software engineering practices, including meeting with clients (the professor, in this case), creating user stories to define what the product does, following Agile principles, using Git to collaborate, and working as part of a team.

Also, Python programming skills and use of the Django framework.
