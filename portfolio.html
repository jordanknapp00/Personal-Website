Personal Projects

These are some of the various projects I've worked on in my spare time over the years. The tables below contain information about each project, as well as download links. Links to my GitHub are available for the major projects, while .zip downloads are available for all projects. Direct downloads for Java projects will include an executable .jar file.

Note that the projects on GitHub do not follow any kind of branching scheme. I didn't use GitHub when developing these projects, and I didn't know about proper GitHub use when initially uploading the projects to GitHub. I'm currently taking a software engineering course, and I now know how to use a develop branch, release branches, feature branches, etc. But I was not able to employ that knowledge when creating the repositories for these personal projects.

These are the major projects, the ones that took a significant amount of time. They could all be described as fully-featured programs, meaning they aren't just quick solutions to a problem.

Smash Bros. Character Picker Java 2019 to 2022 A program which selects random matchups for the game Super Smash Bros. A tier list, defined in a .txt file, is used to determine fair matchups. Matches with up to 8 players are supported. There are systems to prevent duplicate matchups, as well as a built-in soundboard and statistic-tracking system. Other settings are also available to customize the experience. More details can be found in the README file on GitHub.

This was my first time designing a program that is not text-based, and as a result, I had to teach myself how to use the Java Swing library. The result is the largest and most complex project I've worked on, school project or not.

CMSC433: Programming Language Technologies and Paradigms (Spring 2021)

Project Name Language Description Valuable Skills
Restaurant Simulation Java This program simulates the operation of a restaurant. Customers, cooks, and the machines that make the food are all separate, threaded actors that implement the Runnable interface. Customers enter the restaurant and place an order. The order is taken by cooks and processed using the machines, before finally being returned to the customer. The system handles numerous customers, cooks, and machines all operating simultaneously. Java multithreading, including synchronized blocks, semaphores, etc.
Akka Resource Management System Java A program which uses the Akka library to manage actors and their abilities to access resources in a simulated distributed environment. Resources are typically something like a printer, and actors can access and use them using message-passing. Use of Akka and general actor-based concurrency concepts.
Conway's Game of Life with MPI C This program uses the OpenMPI library to create a parallelized version of Conway's Game of Life. Each row of the grid space is run by a different processor, and they use message-passing to communicate data with one another. I used pthreads for C multithreading in an earlier project, but this was my first experience with MPI. Use of the OpenMPI library.

CMSC420: Advanced Data Structures (Spring 2021)

Project Name Language Description Valuable Skills
Traveling Salesperson Tours Java A program which manages tours for the Traveling Salesperson Problem. Three primary data structures are used. First, the tour itself is simply stored in an ArrayList, though it's in a class which features several utilities. A slightly modified AA tree is used to map location names to their index within a tour, which makes looking them up more efficient. Finally, the spatial data, the actual coordinates at which locations are located, is stored in a Wrapped K-D Tree. The Tour and WKDTree classes were written by me. The AAXTree class was provided by the instructor. We had implemented one earlier in the semester, but mine had some errors.

This was a very complicated project, by far the most complicated data structure implementation I've ever done. More details about what exactly this project entailed can be found here. 		Advanced data structure implementation.

CMSC411: Computer Systems Architecture (Fall 2021)

Project Name Language Description Valuable Skills
Pipelined Processor Simulation C A simulation of a basic processor with a simple five-stage pipeline. It runs a simplified MIPS instruction set. The program simulates the registers, as well as both instruction and data memory, with the program producing actual output based on what the instructions do. It has a dependency-checking system, which appropriately stalls the pipeline when necessary.

I think it's safe to say this is the most complex project I'd worked on up to this point, at least in terms of what it set out to do. It was an exceptionally challenging project, but also very rewarding, as it not only honed my programming skills, but also expanded my understanding of how a processor works, even if it only simulates a simple five-stage pipeline and a basic, integer-only instruction set. 		Deeper understanding of how processors work under the hood.
