Name Smash Bros. Character Picker
Language Java
Date 2019 to 2022
Description A program which selects random matchups for the game Super Smash Bros. A tier list, defined in a .txt file, is used to determine fair matchups. Matches with up to 8 players are supported. There are systems to prevent duplicate matchups, as well as a built-in soundboard and statistic-tracking system. Other settings are also available to customize the experience. More details can be found in the README file on GitHub.

This was my first time designing a program that is not text-based, and as a result, I had to teach myself how to use the Java Swing library. The result is the largest and most complex project I've worked on, school project or not.

Note: You can see older versions of this project on GitHub as well, to show the evolution of the program over the couple years I've worked on it. Every major release has been tagged on GitHub. Older versions can be downloaded directly here or using the link to the right.

Name Conway's Game of Life with MPI
Language C
Description This program uses the OpenMPI library to create a parallelized version of Conway's Game of Life. I used pthreads for C multithreading in an earlier project, but this was my first experience with MPI.
Valuable Skills Use of the OpenMPI library.

CMSC420: Advanced Data Structures (Spring 2021)

Name Traveling Salesperson Tours
Language Java
Description A program which manages tours for the Traveling Salesperson Problem. Three primary data structures are used. First, the tour itself is simply stored in an ArrayList, though it's in a class which features several utilities. A slightly modified AA tree is used to map location names to their index within a tour, which makes looking them up more efficient. Finally, the spatial data, the actual coordinates at which locations are located, is stored in a Wrapped K-D Tree. The Tour and WKDTree classes were written by me. The AAXTree class was provided by the instructor. We had implemented one earlier in the semester, but mine had some errors.

This was a very complicated project, by far the most complicated data structure implementation I've ever done. More details about what exactly this project entailed can be found here.
Valuable Skills Advanced data structure implementation.

CMSC425: Game Programming (Spring 2022)

Name Mob Hunt
Language C# (in the context of Unity development)
Description A group project in which I worked with one other person to create a video game called Mob Hunt. In the game, the player traverses levels and sets off traps in order to kill enemies. There are three kinds of enemies, each of which behave somewhat differently. There is a fairly robust AI system, in which enemies can move between patrol, pursuit, and search states.

It should be noted that all art assets came from Kenney, a website that provides free assets for games. I am not an artist. As mentioned, this was a group project. I was responsible for creating animation for those 3D models, however. This was done with a simple sine wave moving limbs back and forth, resulting in Minecraft-esque animations. I also implemented player movement, camera movement (which includes a system for turning objects between the player and camera transparent), enemy AI, parts of the interface, and one of the two levels. Everything else, including the game concept itself, was done by my teammate.

I don't think the game ended up being particularly impressive, but it was definitely a valuable learning experience. We struggled with the concept for the game, so it ended up changing many times throughout the development cycle. If we'd known what we wanted to do from the start, the game may have turned out better.
Valuable Skills C# programming, Unity development, working on a team, and project planning.

CMSC435: Software Engineering (Spring 2022)

Name Online Casino
Language Python (using Django framework), HTML, JavaScript
Description By far the most useful project during my time at UMD, this was a group project in which I developed a web app along with four other people. We followed Agile principles while developing the app, including three sprints and roughly twice-weekly scrum meetings (daily scrum meetings weren't really feasible due to the scale of the project and people's busy schedules). The first sprint was one week, while the subsequent sprints were two weeks each, leading to a five-week development cycle. This was perhaps the most valuable part of the project to me, learning how software is actually developed in the industry. This class also taught me how to make my code more readable and how to use Git to collaborate, both of which were applied during the development of this project.

The project itself was Online Casino, a website in which users can deposit and withdraw "money" in order to gamble in five casino games -- Poker, Blackjack, Craps, Roulette, and Slot Machines. The first three featured fully-functional multiplayer, while the latter two were single-player only. Multiplayer was implemented using websockets, something I had never used before. They ended up being a huge struggle to wrap my head around. Users can sign up, create accounts, become friends with other users, send links to one another to easily join each other's games, and chat with one another during the games. There is a spectating system so people can watch the games unfold without participating. Other minor features include tutorials for each game and a system to limit the amount of money that can be deposited into your account each month.

As this was a group project, obviously I was not responsible for everything. One member of our group had already used Django before, so we tended to follow his lead for just about everything. I was responsible for implementing the depositing and withdrawing money systems, the tutorials, the chat system, and the game of Craps.

As mentioned, I think this was the most valuable project I worked on during my time at UMD. This class and project gave me more exposure to real software engineering practices than every other class combined.
Valuable Skills Software engineering practices, including meeting with clients (the professor, in this case), creating user stories to define what the product does, following Agile principles, using Git to collaborate, and working as part of a team.

Also, Python programming skills and use of the Django framework.
