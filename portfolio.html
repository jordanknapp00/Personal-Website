Major Projects

Major projects are all fully-featured programs, larger in scope than minor projects. These all took a significant amount of time to develop.

Name Smash Bros. Character Picker
Language Java
Date 2019 to 2022
Description A program that selects random matchups for the game Super Smash Bros. A tier list, defined in a .txt file, is used to determine fair matchups. Matches with up to 8 players are supported. There are systems to prevent duplicate matchups, in order to keep things fresh. There is also a built-in soundboard and a statistics tracking system. There are a variety of other settings to customize the experience. More details can be found in the README on GitHub.

This was my first time designing a program that is not text-based, and as a result, I had to teach myself how to use the Java Swing library. The result is one of the largest and most complex projects I've worked on, school project or not.

Note: You can see older versions of this project on GitHub as well, to show the evolution of the program over the couple years I've worked on it. Every major release has been tagged on GitHub. Older versions can also be downloaded directly here or using the link below.

Name Dungeon Generator (AKA dungen)
Language C#
Date April 2022
Description A program that generates simple dungeons in the style of Rogue (1980). The algorithm is based on one provided by my game development professor, which itself was based on the algorithm used in Rogue. I've modified the algorithm a little bit and added the ability to customize the dungeon via command line parameters. I've also added some object-oriented features to the program. I wrote this project attempting to use good software engineering practices (particularly the use of develop and feature branches with Git), in order to build that up as a habit. I also wrote it as a test of my C# skills outside the context of Unity development.

For details on how to use it, see the README on GitHub or the one included with the download.

This program is text-based, designed to be run from the command line.

Minor Projects

Minor projects are small programs, not as fully-featured or wide in scope as major projects. These are mostly text-based programs that quickly solve a problem, test something, or otherwise do something simpler than major projects. These are only available via .zip download. I didn't think they were important enough to warrant putting them on GitHub.

Name Rudimentary Game Engine
Language Java
Date June 2020 to August 2021
Description An unfinished, 2D game engine that I made by following this tutorial. It was meant to power a simple role-playing game. Ultimately, I abandoned this project because I was unhappy with the quality of the tutorial. I instead decided to take a game programming class in my last semester at UMD.

The engine is only capable of displaying a single map, with a single character walking around that map. The collision detection is also somewhat buggy. If this project were completed, or even in a state that might be called playable, I'd probably put it in the major projects category, because it was a lot of work to even get it this far.
